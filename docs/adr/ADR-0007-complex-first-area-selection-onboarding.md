# ADR-0007 단지 우선 검색과 평형 선택 온보딩

## 상태

승인

## 배경

초기 온보딩이 `정확한 주소 -> 동/호 입력 -> Property resolve` 순서로 설계되어 있었다. 하지만 실제 사용자 입력은 대체로 아파트 단지명 수준에서 시작하고, 동/호를 모르는 상태에서도 평형 선택까지 빠르게 진행하고 싶어 한다. 현재 mock 검색 품질도 단지명 중심 탐색에 더 적합하다.

## 결정

온보딩 기본 흐름을 아래 순서로 바꾼다.

1. 단지명 또는 도로명 수준으로 검색한다.
2. 단지 후보를 선택한다.
3. 시스템이 평형 후보를 포함한 `Property` 요약을 resolve 한다.
4. 사용자가 평형 버튼을 고른다.
5. 선택한 평형을 `Property`에 저장하고 이후 도면 확보를 진행한다.

동/호 입력은 기본 흐름에서 제거하고, 향후 정밀 보정이 필요할 때 보조 단계로 추가한다.

## 결과

- 주소 검색 UX가 실제 사용자 입력 패턴과 가까워진다.
- 프로젝트 생성까지 필요한 입력 수가 줄어든다.
- `Property`와 `FloorPlanCandidate`가 평형 기준으로 더 일관되게 연결된다.
- 상세 주소 보정은 별도 phase로 독립시킬 수 있다.

# ADR-0006 스타일 이미지는 mock provider로 먼저 구현한다

## 상태

승인

## 배경

스타일 이미지는 최종적으로 외부 이미지 생성 모델, 프롬프트 템플릿 버전 관리, 비용 제어까지 포함하는 별도 워크플로로 확장될 예정이다.
하지만 현재 단계에서는 프로젝트 문맥과 스타일 preset이 실제 이미지 카드 흐름으로 이어지는지 먼저 검증하는 것이 중요하다.

## 결정

이번 단계의 스타일 이미지는 아래 방식으로 구현한다.

- 이미지 생성은 `StyleImageProvider` 인터페이스 뒤에 둔다.
- 실제 구현은 `MockStyleImageProvider`로 시작한다.
- provider 결과에도 `prompt_text`, `model_name`, `metadata`, `seed`, `storage_key`를 저장한다.
- 좋아요 액션은 `project_style_selections`와 함께 동기 저장한다.

## 이유

- 외부 이미지 API 키나 비용 설정 없이 UI와 persistence 흐름을 먼저 검증할 수 있다.
- prompt 조합 구조를 유지한 채 실제 provider로 교체하기 쉽다.
- 후속 phase에서 선택된 스타일과 공정/전문가 흐름을 연결하기 쉽다.

## 결과

장점:

- 스타일 페이지를 실행 가능한 상태로 빠르게 닫을 수 있다.
- provider 교체 범위를 adapter 계층에 국한할 수 있다.
- 이미지 카드 DTO를 후속 고도화 없이도 검증할 수 있다.

단점:

- 실제 이미지 품질과 생성 시간은 검증하지 못한다.
- mock URL과 metadata는 제품 최종 상태보다 단순하다.

## 후속 작업

- 실제 이미지 생성 provider와 비용/재시도 정책을 붙인다.
- 스타일 선택이 공정 플랜/전문가 추천과 연결되도록 확장한다.

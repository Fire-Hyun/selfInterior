# ADR-0005 Expert 추천은 프로젝트 신호와 seed 카탈로그를 조합해 시작한다

## 상태

승인

## 배경

`ExpertLead`는 최종적으로 평판, 응답 SLA, 실제 제휴 상태, 지역별 공급량까지 반영하는 추천 시스템으로 확장될 예정이다.
하지만 현재 단계에서는 `Project`, `Property`, `ProcessPlan`, `VisualQuestion`에서 이미 확보한 신호를 활용해 실행 가능한 추천 흐름을 먼저 완성하는 것이 중요하다.

## 결정

이번 단계의 전문가 추천은 아래 방식으로 구현한다.

- 전문가 데이터는 migration seed로 초기 카탈로그를 제공한다.
- 추천 계산은 `Project`, `Property`, 최근 `VisualQuestion` 답변을 조합한 내부 service에서 수행한다.
- `HIGH` 이상 risk level은 질문 문맥 기반 우선 카테고리를 먼저 사용한다.
- 그 외에는 현재 공정 단계 기반 기본 카테고리를 사용한다.
- 전문가 목록 노출은 `ACTIVE` 상태만 허용한다.

## 이유

- 외부 제휴 시스템 없이도 추천과 리드 흐름을 즉시 검증할 수 있다.
- 추천 이유를 현재 프로젝트 신호로 설명하기 쉽다.
- 후속 phase에서 실제 파트너 점수와 운영 지표를 붙여도 API 계약을 유지하기 쉽다.

## 결과

장점:

- 프로젝트 홈과 질문 화면에서 전문가 CTA를 실제 데이터로 연결할 수 있다.
- seed 카탈로그만으로도 추천 결과와 리드 생성을 end-to-end로 검증할 수 있다.
- 추천 로직을 service 계층에 집중시켜 후속 고도화 지점을 명확히 남긴다.

단점:

- 추천 품질이 운영 데이터 기반 시스템보다 단순하다.
- 지역과 예산이 빈약한 경우 노출 결과가 제한적일 수 있다.

## 후속 작업

- 전문가 응답 SLA, 운영 승인 상태, 후기 점수 가중치를 반영한다.
- 리드 알림과 후속 상태 변경 이벤트를 비동기 워크플로로 확장한다.

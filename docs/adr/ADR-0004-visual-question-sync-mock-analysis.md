# ADR-0004 VisualQuestion은 동기 mock 분석으로 먼저 완성한다

## 상태

승인

## 배경

사진 질문은 최종적으로 비동기 이미지 업로드, 분석 큐, RAG, 전문가 연결까지 확장될 예정이다.
하지만 현재 단계에서는 `VisualQuestion` 도메인과 UI 흐름을 먼저 닫는 것이 중요하다.

## 결정

`VisualQuestion`은 이번 단계에서 아래 방식으로 구현한다.

- 이미지 저장은 `VisualQuestionStorage` 인터페이스 뒤에 둔다.
- 분석은 `VisionQaClient` 인터페이스 뒤에 둔다.
- 실제 구현은 동기 mock 분석으로 먼저 답변을 생성한다.
- `visual_questions.status`는 유지해 추후 비동기 큐 기반 처리로 교체 가능하게 둔다.

## 이유

- 사용 흐름을 먼저 실행 가능하게 만들 수 있다.
- 파일 저장과 분석을 각각 교체 가능한 adapter 뒤에 둘 수 있다.
- 후속 phase에서 큐와 실제 모델 분석을 붙여도 API 계약을 유지하기 쉽다.

## 결과

장점:

- 사진 질문 등록부터 답변 조회까지 한 턴에 닫힌다.
- mock storage와 mock analysis를 실제 구현으로 교체하기 쉽다.
- 프로젝트 홈 최근 질문 카드와의 연동을 빨리 검증할 수 있다.

단점:

- 실제 지연/재시도/비동기 상태를 아직 검증하지 못한다.
- 분석 품질은 제품 최종 상태보다 단순하다.

## 후속 작업

- 비동기 job과 상태 polling 구조를 추가한다.
- 실제 Vision 모델과 guide/RAG 결합 로직으로 확장한다.
